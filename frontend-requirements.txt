==============================
PROJECT OVERVIEW
==============================
- Detected frontend framework: React 19 (via Vite)
- Primary Libraries: 
    - Tailwind CSS (Styling)
    - motion/react (Animations/Transitions)
    - lucide-react (Iconography)
    - react-router-dom (Routing)
    - shadcn/ui (UI Components)
- Detected routing structure: Browser-based client routing with specialized sub-pages for gaming and social features.
- Detected global layouts: 
    - Sidebar (Desktop fixed/side, Mobile accessible)
    - Navbar (Top-level identity/navigation)
    - BottomNav (Mobile-specific navigation)
    - FriendChatOverlay (Global persistent chat bubbles)
- Detected authentication assumptions:
    - Session-based access for majority of pages (Dashboard, Chat, Profile, etc.)
    - User has attributes: Level (XP-based), Membership (None, PLUS, VIP), Online Status, Badges.
    - Presence is tracked globally and displayed via indicators.

==============================
DETECTED PAGES
==============================
- Landing Page (/) : Platform entry, value proposition, and access points.
- Auth Page (/auth) : Registration and Login handling (Username, Email, Password).
- Dashboard Page (/dashboard) : User hub featuring activity summaries, streak tracking, and personalized recommendations.
- Chat Page (/chat) : Global and game-specific communication channels.
- Lobbies Page (/lobbies) : Browse and search for active game lobbies.
- Lobby Room Page (/lobby/:id) : Interactive room for a specific lobby with voice controls and match management.
- Profile Page (/profile) : Detailed user stats, XP progress, and membership management.
- Games Page (/games) : Discovery and search for compatible games.
- My Games Page (/my-games) : Personal collection management.
- Leaderboard Page (/ranking) : Competitive rankings with reward distribution info.
- Premium Page (/premium) : Comparison and purchase flow for PLUS and VIP tiers.
- Settings Page (/settings) : Account, privacy, and interface configuration.
- Friends Page (/friends) : Comprehensive friend management (Requests, Blocked list, Favorites).

DETECTED GLOBAL SYSTEMS
==============================
- Authentication & Sessions: Handled via Login/Register pages + persistent session for all features.
- User Presence & Status: Comprehensive system tracking Online, Offline, In-Game, In-Lobby, Away.
- Friends & Social Graph: Friendship management (Requests, Acceptance, Blocking, Muting, Favorites).
- Mini Profile System (QuickProfilePopover): Globally available context-aware user view for quick social actions.
- Chat System (Text): Highly interactive multimedia system (GIFs, Reactions, Replies, Mentions, Invites).

- Notification System: Red dot indicators for mentions, invites, and requests.
- Ranking & Rewards: Points-based system linked to active participation.
- Premium / Subscription System: Feature gating for cosmetic enhancements (VIP/PLUS) and priority room access.
- Settings & Preferences: Profile customization (Bio, Banner, Avatar) and platform preferences.

==============================
MISSING OR IMPLICIT BACKEND REQUIREMENTS
==============================
- Real-time Signaling (Socket.io/Websocket): Required for all chat, presence, and lobby state synchronization.
- WebRTC Signaling: Required for Voice status and stream routing (SFU architecture implied).
- Automated Points Engine: Backend needs to hook into "Lobby Create", "Game Start", and "Chat Daily" events to assign Ranking Points.
- Permission Middleware: Verify Host-status for room actions (Start/Kick/Lock).
- Anti-Abuse/Rate Limits: Mentioned in spec for chat (5 msgs/2 sec).
- Pagination: All "View All" lists imply cursor-based or offset pagination.
- CDN / Asset Hosting: Validation of image types and size (2MB) for avatars and banners.
- Search System: Games, Lobbies, and User search (Persian character support specifically).

==============================
FINAL NOTES
==============================
- Language: The frontend is built for Persian users (RTL UI), so the backend must respect Persian character sets (UTF-8) for all user-generated strings (Usernames, Chat, Bios).
- Performance: High volume of animation triggers (Glows/Shimmers) in the chat feed means backend payloads should be optimized/compact.
- Edge Cases: 
    - Handling "Ghost" presence if a user disconnects without a clean lobby-leave.
    - Consistency of countdowns across varying client latencies.
    - Privilege escalation checks for VIP-only room creation.
