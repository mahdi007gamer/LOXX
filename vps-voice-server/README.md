# 🎙️ راهنمای راه‌اندازی سرور اختصاصی صدای LOXX (Mediasoup WebRTC SFU)

این پکیج شامل یک سرور فوق‌سریع پردازش و روتینگ صدا بر روی بستر **WebRTC** است که به منظور کاهش تاخیر به محدوده **30ms الی 80ms** و ارائه کیفیتی مشابه دیسکورد طراحی شده است. 

با انتقال مدیا به این سرور از پورت‌های UDP، مشکل سربار TCP و لگ‌های انباشته‌شده به طور کامل حل خواهد شد. سرور اصلی LOXX فقط وظیفه مدیریت لابی‌ها و دیتابیس را برعهده خواهد داشت و ترافیک صدا مستقیم به این VPS هدایت می‌شود.

## 🚀 مراحل راه‌اندازی بر روی سرور VPS (146.19.212.212)

### ۱. کپی کردن دایرکتوری به VPS
شما می‌توانید کل این فولدر `vps-voice-server` را فشرده کرده و از طریق دستور `scp` یا نرم‌افزارهایی مثل FileZilla/MobaXterm به سرور خود منتقل کنید:
```bash
scp -r ./vps-voice-server root@146.19.212.212:/root/loxx-voice-sfu
```

### ۲. اتصال به سرور از طریق SSH
تنها کافیست با یوزر root یا یوزری با دسترسی sudo به سرور متصل شوید:
```bash
ssh root@146.19.212.212
```

### ۳. ورود به پوشه پروژه و اجرای اسکریپت نصب تمام‌اتوماتیک
اسکریپت `deploy.sh` به صورت هوشمند تمام ابزارهای زیر را روی اوبونتو/دبیان نصب می‌کند:
* پیکربندی فایروال سیستم (بازکردن پورت‌های UDP 40000 تا 49999 و پورت دیتای 4000)
* ابزارهای کامپایلر C++ (G++, GCC, Make, Python3) که برای بیلد کتابخانه Mediasoup روی لینوکس حیاتی هستند.
* نصب آخرین نسخه Node.js LTS (نسخه 20)
* نصب PM2 جهت مدیریت پروسه و بالا نگه‌داشتن همیشگی سرور صوتی
* کامپایل کدهای تایپ‌اسکریپت و استارت اپلیکیشن در PM2 با قابلیت Auto-Start بعد از ریبوت سرور.

برای ران کردن اسکریپت دستورات زیر را اجرا کنید (استفاده از bash باعث می‌شود نیازی به تغییر پرمیشن فایل بعد از هر بار آپدیت گیت نداشته باشید):
```bash
cd /root/loxx-voice-sfu
sudo bash deploy.sh
```

## 🛠️ دستورات پایش کارکرد سرور صوتی (PM2)

کدهای شما با موفقیت در پس‌زمینه با ماژول PM2 اجرا می‌شوند. برای مدیریت آن‌ها می‌توانید از دستورات زیر استفاده کنید:

* **مشاهده مانیتور لحظه‌ای کانال‌ها و ترافیک صوتی:**
  ```bash
  npx pm2 logs loxx-voice-sfu
  ```
* **بررسی وضعیت منابع پردازنده و رم مصرفی سرور:**
  ```bash
  npx pm2 status
  ```
* **ری‌استارت کردن دستی سرور صوتی:**
  ```bash
  npx pm2 restart loxx-voice-sfu
  ```

## 🔒 پیکربندی متغیرهای محیطی در فایل `.env` (اختیاری)
سرور به صورت خودکار با IP اعلام‌شده به عنوان آدرس پیش‌فرض پیکربندی شده است. در صورت نیاز به تغییر پورت لیسینر یا IP دیتای اعلامی، می‌توانید یک فایل `.env` در مسیر ریشه پوشه پروژه بر روی VPS بسازید:
```env
PORT=4000
PUBLIC_IP=146.19.212.212
LISTEN_IP=0.0.0.0
```
بعد از هرگونه تغییر فایل `.env` بر روی سرور، دستور زیر را جهت اعمال تغییرات اجرا کنید:
```bash
npx pm2 restart loxx-voice-sfu --update-env
```

صداها اکنون کاملاً آماده‌ی سرویس‌دهی روی پروتکل بسیار سریع UDP بر روی پورت‌های ۴۰۰۰۰ الی ۴۹۹۹۹ هستند! 🎧🔥
