LOXX Realtime WebSocket Event Contract
Version: 1.0 (Enterprise Edition)
Compatible with: Backend Blueprint v1.0, REST API Contract v1.1
Transport: Socket.IO (namespaces + rooms)
Encoding: UTF-8 (Full Persian/RTL support for strings)

================================================================================
1. CONNECTION & AUTHENTICATION
================================================================================

- Base URL: wss://api.loxx.ir
- Handshake Protocol:
    * Query Parameter: `?token=Bearer <JWT>`
    * OR Header: `Authorization: Bearer <JWT>`
- Handshake Result:
    * Server returns a connection success with `wsVersion: "1.0"`.
    * Client joins a private user room: `user:{id}` automatically.
- Reconnection Policy:
    * Exponential backoff.
    * On `connect_error` due to `AUTH_EXPIRED`, client MUST refresh via REST POST /auth/refresh before retrying WebSocket.

================================================================================
2. COMMON WRAPPERS (ENVELOPES)
================================================================================

All events are wrapped in a standard Envelope.

```typescript
interface EventEnvelope<T> {
  reqId?: string; // UUID for client acknowledgment correlation
  data: T;
  ts: number;    // Server-side timestamp (milliseconds)
}

interface ErrorEnvelope {
  code: 
    | "VALIDATION_FAILED"
    | "AUTH_EXPIRED"
    | "INSUFFICIENT_PERMISSIONS"
    | "RESOURCE_NOT_FOUND"
    | "LOBBY_FULL"
    | "RATE_LIMIT_EXCEEDED"
    | "INTERNAL_ERROR"
    | "SERVICE_UNAVAILABLE";
  message: string; // Human readable error (Persian supported)
  ts: number;
}

interface AckResponse<T = any> {
  reqId: string;
  status: "ok" | "error";
  data?: T;
  error?: ErrorEnvelope;
}
```

3. NAMESPACE: /presence
================================================================================

Handles user online status and real-time activity broadcasting.

- Client Emit: `presence.update`
  * Payload: `{ status: "online" | "offline" | "away" | "in_lobby" | "in_game", activity?: string }`
  * Rate Limit: 10 events / minute / user.
  * Side Effect: Updates `Profile.lastActivity`.

- Server Emit: `presence.snapshot`
  * Target: Individual user on join.
  * Payload: `{ users: Array<{ userId: string, status: string, activity?: string }> }`
  * Context: Initial state of online friends.

- Server Emit: `presence.changed`
  * Target: Broadcasted to the private rooms of all mutual "ACCEPTED" friends.
  * Payload: `{ userId: string, status: string, activity?: string }`

================================================================================
4. NAMESPACE: /lobby
================================================================================

Handles the real-time state of gaming rooms.

- Client Emit: `lobby.join`
  * Payload: `{ lobbyId: string, password? : string }`
  * Ack Data: `{ lobbyId: string, members: Array<{ userId: string, role: string, isReady: boolean, micMuted: boolean }>, you: { role: string, isReady: boolean, micMuted: boolean } }`
  * Logic: Joins user to socket room `lobby:{lobbyId}`.

- Client Emit: `lobby.leave`
  * Payload: `{ lobbyId: string }`

- Client Emit: `lobby.ready`
  * Payload: `{ lobbyId: string, ready: boolean }`
  * Idempotency: If state is identical to DB `LobbyMember.isReady`, server acks "ok" but does NOT broadcast update.
  * Rate Limit: 30 events / minute.

- Client Emit: `lobby.mic`
  * Payload: `{ lobbyId: string, muted: boolean }`
  * Side Effect: Updates `LobbyMember.micStatus`.

- Server Emit: `lobby.member_joined`
  * Target: Room `lobby:{id}`
  * Payload: `{ user: { id: string, username: string, role: string, avatarUrl?: string }, membersCount: number }`

- Server Emit: `lobby.member_left`
  * Target: Room `lobby:{id}`
  * Payload: `{ userId: string, membersCount: number }`

- Server Emit: `lobby.member_updated`
  * Target: Room `lobby:{id}`
  * Payload: `{ userId: string, isReady?: boolean, micMuted?: boolean }`

- Host Actions (Client Emit):
  * `lobby.kick`: `{ lobbyId: string, targetUserId: string }`
  * `lobby.transfer_host`: `{ lobbyId: string, targetUserId: string }`
  * Note: Requires `LobbyMember.role == "HOST"`.

5. NAMESPACE: /chat
================================================================================

High-frequency text and invite channel.

- Client Emit: `chat.send`
  * Payload: `{ target: { type: "channel" | "lobby", id: string }, content: string, tempId: string, replyToId?: string }`
  * Anti-Spam: 5 messages / 5 seconds sliding window.
  * Ack Data: `{ tempId: string, messageId: string, createdAt: number }`
  * Rule: Max content length 1000 characters. Persian supported.

- Server Emit: `chat.message`
  * Target: Room `channel:{id}` or `lobby:{id}`
  * Payload: `{ messageId: string, from: { userId: string, username: string, membership: string }, content: string, metadata?: any, createdAt: number }`

- Server Emit: `chat.history`
  * Target: Individual user on room join.
  * Payload: `{ items: Array<Message>, nextCursor?: string }`

================================================================================
6. NAMESPACE: /notify
================================================================================

Persistence markers and real-time alerts.

- Server Emit: `notify.new`
  * Target: User's private room `user:{id}`
  * Payload: `{ id: string, type: "FRIEND_REQUEST" | "MENTION" | "LOBBY_INVITE", data: any, createdAt: number }`

- Client Emit: `notify.read`
  * Payload: `{ ids?: string[], all?: boolean }`
  * Ack: `{ status: "ok" }`

================================================================================
7. NAMESPACE: /ranking
================================================================================

- Server Emit: `ranking.tick`
  * Target: Subscribed clients.
  * Payload: `{ period: "weekly" | "monthly" | "all-time", topUsers: Array<{ rank: number, username: string, points: number, trend: "up" | "down" | "stable" }> }`
  * Throttle: Broadcasted every 15 seconds max.

8. SAMPLE CLIENT CODE (TypeScript)
================================================================================

```typescript
import { io, Socket } from "socket.io-client";

const socket: Socket = io("wss://api.loxx.ir/lobby", {
  auth: { token: "Bearer my.jwt.token" }
});

// Handling a Lobby Join
const joinLobby = (id: string) => {
  const reqId = crypto.randomUUID();
  socket.emit("lobby.join", { lobbyId: id, reqId }, (ack: AckResponse) => {
    if (ack.status === "ok") {
      console.log("Joined lobby:", ack.data.lobbyId);
    } else {
      console.error("Error joining lobby:", ack.error?.message);
    }
  });
};

// Global Error Handler
socket.on("error", (err: ErrorEnvelope) => {
  if (err.code === "AUTH_EXPIRED") {
    // Initiate token refresh flow
  }
});

// Listening for system messages in Persian
socket.on("chat.message", (envelope: EventEnvelope<any>) => {
  console.log(`${envelope.data.from.username}: ${envelope.data.content}`);
});
```

9. VERSIONING & PERFORMANCE RULES
================================================================================

1. Version Validation: Handshake contains `wsVersion`. If mismatch detected on 
   backend update, server will emit `error` with code `VALIDATION_FAILED` 
   instructing a hard page refresh.
2. Binary Support: Chat reactions and GIF metadata are passed as optimized 
   JSON. Large assets (avatars) are never passed through WebSockets; only 
   URLs provided.
3. RTL Enforcement: Content in `chat.message` is delivered raw; frontend 
   application MUST apply `dir="rtl"` based on UTF-8 character detection or 
   global settings.
